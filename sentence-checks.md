# Sentence-level checks

Apply to standalone sentences and local passages. **Run every objective below in order.** Name each objective, state how it applies, and report issues. If one does not apply, say **not applicable** and why.

Do not skip, merge, or abbreviate objectives.

| # | Objective |
|---|-----------|
| 1 | **Clarify references** — every pronoun (“it”, “which”, “this”, …) points to a specific noun; short back-references tie unambiguously to the concept they invoke. |
| 2 | **Define or replace undefined terminology** — non-standard terms are defined on first use or replaced with standard equivalents. Do not coin labels when a plain description suffices (e.g. “operator that annihilates the code space” vs “null branch”). Name a term only if standard in the field or if it recurs enough to warrant a definition (then define on first use). |
| 3 | **Fix subject–verb–object mismatches** — grammatical agreement and semantic fit, including inside clauses (e.g. states do not “yield” results; a model does not “prove” a conclusion). |
| 4 | **Streamline narrative** — coherent flow between sentences; remove redundant phrases or nouns that signal convoluted logic. |
| 5 | **Polish non-native wording** — concise, idiomatic English; preserve meaning and technical accuracy. |
| 6 | **Balance rigor with narrative** — state physical/mathematical facts as givens, not conditional premises (“Because X is Y, …” not “If X is Y, then …” when X is established). |
| 7 | **Minimal changes** — edit only what clarity and flow require; substantial reordering only with explicit user instruction. |
| 8 | **Prefer active voice** unless passive improves clarity. |
| 9 | **Rewrite “For A, it does B”** — find this structure and rewrite as “A does B”. |
| 10 | **Let physics tell the story** — give math objects physical meaning where possible; avoid math that reads detached from the physics question. |
